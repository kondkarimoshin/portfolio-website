## 📌 Versioning Strategy

This project follows **Semantic Versioning (SemVer)**.

| Type | Example | Description |
|------|---------|-------------|
| **Major** | `2.0.0` | Significant architectural changes, major milestones, or breaking changes. |
| **Minor** | `2.1.0` | New features and enhancements that remain backward compatible. |
| **Patch** | `2.1.1` | Bug fixes, documentation updates, and minor improvements. |
